Running your system against a simulator

Beginning a dialog

POST: /run

  • Request:
    • {
    'task_id': 'TASK ID',
    'key': 'YOUR API KEY',
    'system_id': 'SHORT IDENTIFIER OF YOUR SYSTEM',
    'data': 'WHAT TO SEND TO THE SIMULATOR'
}
  • Response:
  • {
    'status': 1,
    'data': 'SIMULATOR RESPONSE',
    'dialog_key': u'IDENTIFIER FOR THE DIALOG'
}

Continuing a dialog

POST: /run

  • Request: 
    {
    'task_id': 'TASK ID',
    'key': 'YOUR API KEY',
    'system_id': 'SHORT IDENTIFIER OF YOUR SYSTEM',
    'data': 'WHAT TO SEND TO THE SIMULATOR'
}
  • Response:
  • {
    'status': STATUS,
    'data': 'SIMULATOR RESPONE',
    'dialog_key': u'IDENTIFIER FOR THE DIALOG'
}

Providing a simulator

We will call the URL endpoint that you provide to the platform by a HTTP POST request.

  • Request: 
    {
    'dialog_key': DIALOG ID,
    'data': 'DIALOG SYSTEM REQUEST'
}
  • Expected response: 
    {
    'data': 'DIALOG SYSTEM REQUEST'
}
